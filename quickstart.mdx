## Prerequisites

Before you begin, ensure you have:

- Python 3.11+
- Node.js 18+ (for Admin Console)
- PostgreSQL database with pgvector extension
- Redis instance
- Firebase project (for authentication)
- Google Cloud Platform account

## Local Development

### 1. Clone the Repository

```bash
git clone https://github.com/sari-platform/sari.git
cd sari
```

### 2. Set Up the API Backend

```bash
cd mcp-backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
```

Edit `.env` with your configuration:

```bash
# Database
CLOUD_SQL_CONNECTION_NAME=your-project:region:instance
DB_NAME=sari_db
DB_USER=app_user

# GCP
GCP_PROJECT_ID=your-project-id
USE_SECRET_MANAGER=false  # Set to true in production

# Redis (optional for local dev)
REDIS_HOST=localhost
REDIS_PORT=6379

# Features
ENABLE_VECTOR_EMBEDDINGS=true
ENABLE_EMAIL_NOTIFICATIONS=false
```

Start the backend:

```bash
uvicorn main:app --reload --port 8080
```

<Check>
  API Backend is now running at `http://localhost:8080`
</Check>

### 3. Set Up the MCP Server

```bash
cd ../dynamic-openapi-mcp

# Create virtual environment
python -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
export MCP_TRANSPORT=stdio
export VECTOR_SEARCH_API_URL=http://localhost:8080/api/vector/search
export OPENAPI_SPEC_URL=http://localhost:8080/api/collections
export RATE_LIMIT_ENABLED=false
```

### 4. Set Up the Admin Console

```bash
cd ../api-vault-guard

# Install dependencies
npm install

# Configure environment
cp .env.example .env
```

Edit `.env`:

```bash
VITE_API_URL=http://localhost:8080
VITE_FIREBASE_API_KEY=your-firebase-api-key
VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
VITE_FIREBASE_PROJECT_ID=your-project-id
```

Start the dev server:

```bash
npm run dev
```

<Check>
  Admin Console is now running at `http://localhost:5173`
</Check>

## Verify Installation

### Health Checks

```bash
# API Backend
curl http://localhost:8080/health
# Expected: {"status": "healthy", "service": "mcp-backend"}

# Vector Search
curl http://localhost:8080/api/vector/health
# Expected: {"status": "healthy", "embedding_enabled": true}
```

### Test Authentication

1. Open the Admin Console at `http://localhost:5173`
2. Sign up for a new account
3. Check the database for the pending registration
